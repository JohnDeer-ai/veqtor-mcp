# Synthetic activation corpus

These four DOCX files are generated by Veqtor's deterministic public fixture
generator during the release build. They contain no client data.

Use `FIRST_PROMPT.txt` to check that Claude Desktop can see the extension and
call its read tools. This first prompt is intentionally read-only. Filename
order is deterministic navigation only; it is not proof that one document was
derived from another.

Some source revisions use the author value `53` deliberately. It is an unusual
but valid raw Word author string used to prove that Veqtor preserves source
metadata without inventing or verifying a person's identity.

The DOCX files are immutable release inputs. Veqtor may add a private `.veqtor`
sidecar when a tool is called, but it does not modify the four source files.
Never create a write-test output in this bundled folder. Release acceptance
copies only these four DOCX files to a fresh writable folder outside the
installed extension, writes the new output there, then uses `list_rounds` and
`extract_redlines` to confirm the source hash stayed unchanged and the output
hash agrees with `apply_edits`.
